The format transition
Legacy format
ESA SAFE / ZIP
  • File-based archive (ZIP + XML)
  • Full download required
  • Mission-specific structure
  • Not cloud-optimised
→
New format
EOPF Zarr
  • Chunked, cloud-native arrays
  • Lazy access — no full download
  • Harmonised across missions
  • Works directly with xarray, Python

Code access

Access the data programmatically

Recommendation

Work in the cloud via JupyterHub whenever possible. Zarr's chunked format is designed for lazy remote access — direct download is supported but discouraged.

import xarray as xr # Open EOPF Zarr product directly from object store ds = xr.open_dataset( "s3://e05ab01a9d56408d82ac32d69a5aae2a:202506-s02msil2a/...", engine="eopf-zarr", chunks="auto" ) ds["B04"].isel(time=0).plot()
from pystac_client import Client # Search the STAC catalogue catalog = Client.open("https://stac.browser.user.eopf.eodc.eu") items = catalog.search( collections=["sentinel-2-l2a"], bbox=[12.0, 41.5, 13.0, 42.5] ).item_collection()
s5cmd \ --credentials-file ~/.aws/credentials \ cp \ s3://e05ab01a9d56408d82ac32d69a5aae2a:202506-s02msil2a/*.zarr \ ./local_output/

Open-source plugins

Tools & integrations

Use EOPF Zarr products inside your existing toolchain. All plugins are open-source and maintained on GitHub.

Python
xarray-eopf

xarray backend for reading EOPF products. Install with pip and pass engine="eopf-zarr" to any xr.open_dataset() call.

Python
xcube data store

Adds an eopf store to xcube for analysis-ready datacube (ARDC) access. Pairs well with the JupyterHub environment.

GDAL
GDAL reader plugin

Open, process, and visualise EOPF Zarr products in any GDAL-compatible GIS tool. Community-driven and open-source.

Julia
Julia reader

Lightweight direct access to EOPF Zarr products for high-performance numerical workflows in Julia.

SNAP
SNAP reader (prototype)

Prototype plugin for ESA's SNAP toolbox. Supports Sentinel-2 L2A Zarr products. Demonstrates desktop toolchain compatibility.

About

What is EOPF Zarr?

The Earth Observation Processing Framework (EOPF) is ESA's initiative to modernise how Copernicus Sentinel data is stored, distributed, and processed.

The technology

Why Zarr?

What Zarr is

Zarr is a community-maintained, open format for storing large, chunked N-dimensional arrays. It is the cloud-native successor to NetCDF and an alternative to COG that handles multi-dimensional data more naturally.

Why ESA chose it

Zarr supports lazy I/O — users access only the spatial or temporal slice they need, with no full download. This makes it ideal for object store deployment and directly compatible with xarray, Dask, and NumPy.

What EOPF adds

EOPF defines a common metamodel layered on top of Zarr, harmonising product structure across all Sentinel missions. A single code pattern opens S1, S2, or S3 products without mission-specific workarounds.

Where this is going

This service is an early-access proving ground. The full transition of operational Copernicus products is funded and in progress. The service exists to validate the format with real users before the full rollout.

Architecture

How data is reformatted

Existing CDSE products are converted automatically via the EOPF CPM library and published to a STAC catalogue.

Step 1
Query CDSE

Download SAFE products over a configurable area of interest

Step 2
Convert

EOPF CPM library reformats SAFE → Zarr

Step 3
Upload

Zarr stores written to EODC S3 object storage

Step 4
Catalogue

STAC items generated and registered in the API

Security

The STAC API and STAC Browser run behind an API Gateway. Authentication is handled via Copernicus Data Space Ecosystem (CDSE) identities.
